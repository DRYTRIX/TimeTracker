An include is rendered as a separate template, so a child template's override cannot reach a block declared here -- page titles and per-page CSS would silently fall back to the defaults. All such declarations stay in base.html. #} {# Font Awesome, CSS build only. The SVG-with-JS build (6.4.2) used to be loaded in addition to this one; the two conflicted and doubled the icon payload for no benefit. No template uses the JS-only APIs (fa-layers, data-fa-transform, fa-mask), so the CSS build is sufficient. MUST load BEFORE dist/output.css. Font Awesome declares `.fa-solid,.fab,.far,.fas { display: var(--fa-display, inline-block) }`, which has the same specificity (0,1,0) as Tailwind's `.hidden { display: none }`. Source order therefore decides the winner, and Tailwind is not configured with the `important` strategy. Loading Font Awesome last makes every `hidden` on an icon a no-op — 27 elements including the theme-toggle icons. #} {# Inter is self-hosted: @font-face rules in app/static/src/input.css point at app/static/vendor/inter (previously an @import from fonts.bunny.net). #} {# Per-user custom theme (see app/services/theme_service.py).
