1) Deploy the Peppol Bridge

TimeTracker sends UBL XML to a local bridge. The bridge forwards it to your Access Point provider (e.g. Peppyrus or e-invoice.be).

This selection affects the suggested bridge environment variables below.

This token is used by TimeTracker as PEPPOL_ACCESS_POINT_TOKEN.

Paste this into your compose file. Then set the provider API key in your .env.

2) Connect TimeTracker to the Bridge

These settings enable the existing Generic Peppol transport and point it at the bridge.

If your bridge runs as a compose service named peppol-bridge, this works out of the box.

3) Send a test invoice

Pick a recent invoice and attempt a real Peppol send using the current configuration. The invoice must have a linked client with custom_fields.peppol_endpoint_id and custom_fields.peppol_scheme_id configured.
